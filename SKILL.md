# BTC 短线 50 EMA 突破策略 V6.0

## 执行节奏
**每 15 分钟自动触发一次**  
（适合通过 Cron Job、定时任务或 Agent 循环每15分钟运行一次，确保不重叠执行）

## Step 0 · 准备工作
1. 调用 `market_get_ticker` 获取 BTC-USDT-SWAP 最新实时价格  
2. 调用 `swap_set_leverage` 设置杠杆为 4，模式 `cross`  
3. 调用 `swap_get_positions` 检查持仓  
   - 如果存在任何持仓（数量 > 0），则立即跳过本轮，输出“存在持仓，本轮禁止开新单”  
4. 调用 `account_balance` 查询可用余额

## Step 1 · 数据采集
- 调用 `market_get_candles` 获取 **15分钟 K 线最近 100 根**（主框架，用于EMA判断、回调、水平线）  
- 调用 `market_get_candles` 获取 **1小时 K 线最近 50 根**（大趋势辅助，可选过滤：仅当1H趋势与信号同向时考虑）

## Step 2 · 50 EMA 计算
调用 `market_get_indicator` 获取 50 EMA：  
- `instId = "BTC-USDT-SWAP"`  
- `indicator = "ema"`  
- `bar = "15m"`  
- `params = "50"`  

**提取**：EMA50 值 → 核心趋势分界线

## Step 3 · 短线入场逻辑

### 做多信号（Long）
- 价格**长期在50 EMA下方运行**（最近20根15m K线中，至少15根收盘 < EMA50）  
- **突然突破并收盘在上方**（最近1根K线实体收盘 > EMA50，且前一根 < EMA50）  
- 等待至少**2根连续阴线（红/跌K）回调**（pullback确认）  
- 在突破前的波段高点画水平线（即回调前那根突破K线的高点或前高）  
- 价格**再次向上突破该水平线**（K线实体收盘在上方）→ **入场做多**  

**无效条件**（任一触发即跳过）：  
1. 回调期间价格再次跌破50 EMA  
2. 初始突破幅度过大（>3-4倍正常K线实体，易反转）

### 做空信号（Short）
- 价格**长期在50 EMA上方运行**（最近20根15m K线中，至少15根收盘 > EMA50）  
- **突然跌破并收盘在下方**（最近1根K线实体收盘 < EMA50，且前一根 > EMA50）  
- 等待至少**2根连续阳线（绿/涨K）回调**（pullback确认）  
- 在突破前的波段低点画水平线（即回调前那根突破K线的低点或前低）  
- 价格**再次向下突破该水平线**（K线实体收盘在下方）→ **入场做空**  

**无效条件**同上（对称）

**否则**：跳过

## Step 4 · 执行下单
只有**无持仓且信号触发**时，立即执行下单：  
- `instId = "BTC-USDT-SWAP"`  
- `side = <buy 或 sell>`  
- `ordType = "market"`  
- `posSide = <long 或 short>`  
- `tdMode = "cross"`  
- `sz = <根据可用余额计算的合理张数（单笔风险控制在账户2-3%）>`  
- `tag = "agentTradeKit"`

## Step 5 · 止盈止损（短线风格）
- **止损**：Chandelier Stop指标放在入场点下方（若API不支持，可用固定开仓价 ±1% 替代）  
- **止盈**：2倍止损距离（严格1:2 风险回报比）  
- 数量 = 全部持仓（或固定比例，一次性平仓）

## 风控规则
- 有持仓时严格禁止开新单  
- 单笔风险控制在账户 2-3%  
- 杠杆仅4倍  
- 严格遵守无效条件，避免假突破

## 输出要求
必须首先输出：  
1. 当前真实 BTC 价格  
2. 当前持仓情况  
3. 50 EMA数值 + 当前价格距离（%）  
4. 15分钟K线形态判断（含EMA位置、回调K线、水平线突破情况）  
5. 最终决策及工具调用详情（包含订单ID、成交信息）
